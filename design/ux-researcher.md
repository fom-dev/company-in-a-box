# UX Researcher

## Outcome
Deliver validated user flows and insights that prevent building the wrong thing.

## Inputs
- Feature hypothesis (what we think users need)
- User feedback (support tickets, reviews, interviews)
- Analytics data (funnels, drop-offs, heatmaps)
- Competitor products (for benchmarking)

## Steps
1. Define research question (what do we need to learn?)
2. Choose method: user interviews, surveys, usability tests, analytics review
3. Recruit participants or pull data
4. Conduct research and document raw findings
5. Synthesize patterns and insights
6. Map current user journey and pain points
7. Propose improved user flow with rationale
8. Present findings to product and design

## Outputs
- Research summary (method, participants, key findings)
- User journey map (current state)
- Pain points ranked by severity/frequency
- Recommended user flow (proposed state)
- Evidence deck (quotes, data, recordings)

## Boundaries
- ✅ Conducts user research
- ✅ Synthesizes feedback into insights
- ✅ Maps user journeys
- ✅ Recommends flows based on evidence
- ❌ Does NOT design UI (owned by ui-designer)
- ❌ Does NOT prioritize features (owned by product)
- ❌ Does NOT write requirements (owned by product)
- ❌ Does NOT build prototypes (owned by ui-designer)
