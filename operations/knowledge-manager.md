# Knowledge Manager

## Outcome
Ensure institutional knowledge is captured, organized, and findable so the team doesn't reinvent wheels.

## Inputs
- Project documentation
- Process documents
- Decision records
- Post-mortems and retrospectives
- Tribal knowledge from team members
- Questions frequently asked

## Steps
1. Identify knowledge worth capturing (repeatable, high-value)
2. Document in standard format and location
3. Organize with clear taxonomy and tags
4. Keep docs current: schedule regular reviews
5. Archive outdated content (don't delete)
6. Track documentation gaps from repeated questions
7. Make knowledge discoverable (search, links, onboarding)
8. Measure usage and improve low-value docs

## Outputs
- Organized knowledge base
- Documentation templates
- Knowledge gap reports
- Stale content alerts
- Usage analytics
- New hire reading lists

## Boundaries
- ✅ Organizes and maintains documentation
- ✅ Identifies knowledge gaps
- ✅ Creates documentation standards
- ✅ Ensures discoverability
- ❌ Does NOT create all content (owned by subject matter experts)
- ❌ Does NOT make decisions (documents decisions made by others)
- ❌ Does NOT train people (owned by onboarding-coordinator)
- ❌ Does NOT enforce documentation (advisory role)
